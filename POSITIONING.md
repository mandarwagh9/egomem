## One-liner

**Before** (data-labeling framing — reads like a Scale/Surge clone in a crowded market):
> "Black Robotics turns egocentric human task videos into signed, schema-validated,
> training-ready datasets for robotics."

**After** (picks-and-shovels for embodied AI; the neutral-layer outcome is implied):
> **"Black Robotics is the neutral data + memory layer for embodied AI: we turn cheap
> egocentric human video into signed, training-ready datasets — and a model-agnostic
> memory that any world model or robot policy can query, without locking the buyer to one
> model."**

Shorter, for a cold email subject / verbal:
> **"Picks-and-shovels for robot foundation models — the buyer-side data + memory layer."**

Why it's better: it names the *outcome* (own the neutral infrastructure of the embodied-AI
wave) instead of the *task* (labeling), and it states the wedge that separates you from the
data-labeling pack — neutrality + a memory layer nobody else ships.

## Thesis / RFS alignment

State explicitly: **"Software for Agents"** (memory is the canonical agent-infra rail) +
the embodied-AI / robot-foundation-model data wave. Frame Black Robotics as **the data &
memory substrate** for that wave — the picks-and-shovels position that wins regardless of

## Traction framing (how to present whatever you have)

- Lead with the **highest real unit you have**, in this order: paid pilot > design partner/LOI
  > data volume (hours/episodes) + its growth rate > contributors > shipped product.
- **Rate over absolute:** "`[X]` hours of egocentric data, growing `[Y]`% week-over-week" beats
  a static number. Same for contributors.
- **Name durability:** if any interest is from outside your own network (a real robotics
  lab/buyer), say so — it's the single most persuasive line you can write.
- **EgoMem is R&D proof, not traction** — present it as evidence of *technical depth and the
  thesis being real*, never as revenue. Honesty here compounds; spin collapses in DD.

## Anti-signals to fix

- **"Just another data-labeling shop (Scale/Surge/Karya for robots)."** → Reframe on
  **neutrality + the memory layer**: you're not selling labels, you're selling the buyer-side
  layer that makes egocentric data *usable by any model* — a thing the labeling pack and the
  model labs both structurally won't build.
- **"Pre-revenue / thin traction."** → Foreground the shipped stack + the validated wedge +
  the one insight; commit to a near-term revenue/pilot milestone. Go get 1 outside pilot.
- **"India-based, no emerging-market precedent."** → Turn it into the moat: opt-in,
  DPDP-compliant, lower-cost-per-hour egocentric capture at scale is *why* you can build the
  dataset others can't. Pair with SF-willingness if true.
- **"Memory layer = a feature, not a company."** → Correct: the company is **data + memory +
  trust**; EgoMem is the wedge and credibility, not the whole P&L. Say it before they think it.
